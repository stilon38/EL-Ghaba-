# الغابة · التعرّف على الأشخاص (EL-Ghaba Recognition)

تطبيق ويب للتعرّف على الأشخاص من **الوجه** و**الصوت** — يعمل بالكامل داخل متصفحك،
بدون أي خادم، ويتعامل مع **مئات الأشخاص** عبر قاعدة بيانات محلية.

> A privacy-first, in-browser web app that recognizes people by **face** and **voice**.
> Everything runs locally in the browser — no server, no cloud, no data leaves the device.

---

## ✨ المميزات

- 🔍 **تعرّف مباشر على الوجوه** من الكاميرا مع رسم مربّعات وأسماء وثقة التطابق في الوقت الحقيقي.
- 🎙️ **تعرّف على المتحدث بالصوت** عبر بصمة صوتية مبنية على معاملات MFCC.
- ➕ **تسجيل الأشخاص**: التقاط عدة عيّنات وجه + تسجيل بصمة صوت لكل شخص.
- 👥 **إدارة قاعدة البيانات**: عرض، حذف، وتصدير/استيراد نسخة احتياطية (JSON).
- 🔒 **خصوصية كاملة**: كل البيانات تُخزَّن محلياً في متصفحك (IndexedDB). لا يُرسَل أي شيء للإنترنت.
- 📱 يعمل على الهاتف والكمبيوتر، وواجهة عربية بالكامل (RTL).

## 🧠 كيف يعمل؟

| الوسيلة | التقنية | التفاصيل |
|--------|---------|----------|
| الوجه | [`@vladmandic/face-api`](https://github.com/vladmandic/face-api) (TensorFlow.js) | كشف الوجه + استخراج متجه مميّز 128 بُعد، ومطابقة بأقرب جار (مسافة إقليدية). |
| الصوت | [Meyda](https://meyda.js.org/) + معالجة خاصة | بصمة صوت عالية الدقة (85 بُعد): MFCC + مشتقاتها Delta + إحصائيات النبرة (F0) + ميزات طيفية، مع **توحيد قياسي عام (z-normalization)** ومطابقة بأقرب جار وثقة قائمة على الهامش. |
| التخزين | IndexedDB (`idb`) | يتّسع لمئات الأشخاص محلياً مع دعم النسخ الاحتياطي. |

النماذج مضمّنة داخل `public/models` (~13MB) فيعمل التطبيق **دون اتصال بالإنترنت** بعد أول تحميل.

## 🚀 التشغيل

```bash
npm install          # تثبيت الاعتماديات
npm run dev          # تشغيل بيئة التطوير على http://localhost:5173
```

للبناء للإنتاج:

```bash
npm run build        # يُنتج مجلد dist/ جاهز للنشر
npm run preview      # معاينة النسخة المبنية
```

> ⚠️ الكاميرا والميكروفون يتطلبان سياقاً آمناً: `localhost` أو موقعاً عبر **HTTPS**.

إذا احتجت لإعادة نسخ النماذج يدوياً بعد `npm install`:

```bash
npm run fetch-models
```

## 📖 طريقة الاستخدام

1. **تسجيل**: افتح تبويب «تسجيل»، اسمح بالكاميرا/الميكروفون، التقط 3–5 عيّنات وجه بزوايا مختلفة، وسجّل بصمة الصوت، ثم أدخل الاسم واحفظ.
2. **تعرّف**: افتح تبويب «تعرّف» لرؤية التعرّف المباشر على الوجوه، أو اضغط «استمع وتعرّف» للتعرّف بالصوت.
3. **الأشخاص**: أدر قاعدة البيانات وصدّر/استورد نسخة احتياطية.

## 🗂️ البنية


## ⚙️ ملاحظات حول الدقة

- **الوجه** دقيق جداً (نموذج مُدرَّب). زد عدد عيّنات كل شخص لتحسين الثبات مع تغيّر الإضاءة والزاوية.
- **الصوت** يستخدم بصمة غنية (MFCC + Delta + نبرة + ميزات طيفية) مع توحيد قياسي عام عبر كل الأشخاص المسجّلين، ما يُبرز الأبعاد المميِّزة ويرفع دقة التمييز بين **العديد من المتحدثين**. الثقة تُحسب من الفارق (الهامش) بين المرشّح الأول والثاني، ما يقلّل القبول الخاطئ للأصوات الغريبة.
- لأفضل دقة صوتية: سجّل **2–3 عيّنات** لكل شخص بجُمل مختلفة في مكان هادئ. يمكن ضبط العتبات (`VOICE_THRESHOLD`, `VOICE_MIN_MARGIN`) في `src/lib/match.ts`.
- ملاحظة تقنية: هذه طريقة كلاسيكية قوية تعمل بالكامل في المتصفح دون نماذج ثقيلة؛ وهي أخف من نماذج بصمة الصوت العميقة (x-vector/ECAPA) لكنها كافية لعشرات ومئات المتحدثين في بيئة معقولة.

## 🔒 كلمة المرور (بوابة الدخول)

يفتح التطبيق خلف بوابة كلمة مرور. غيّر الكلمة قبل النشر من الملف `src/config.ts`:

```ts
export const APP_PASSWORD = 'الغابة2026'; // ← غيّرها
```

> هذه حماية خفيفة على مستوى المتصفح تمنع الوصول العابر. لحماية حقيقية على مستوى الخادم استخدم **Cloudflare Access** (مجاني) بعد النشر على Cloudflare Pages.

## 🚀 النشر كرابط خاص (يعمل على الآيفون)

الكاميرا/الميكروفون على الآيفون تتطلب **HTTPS**، لذا نحتاج استضافة ثابتة. الأسهل:

### أ) Netlify Drop (من كمبيوتر — الأسرع)
1. شغّل `npm run build` فيتكوّن مجلد `dist/`.
2. افتح <https://app.netlify.com/drop> واسحب مجلد `dist` كاملاً.
3. تحصل فوراً على رابط HTTPS تفتحه من Safari على الآيفون، ويمكنك «إضافته إلى الشاشة الرئيسية» ليصبح كأيقونة تطبيق.

### ب) Cloudflare Pages (من الهاتف أيضاً + حماية أقوى)
1. اربط المستودع (اجعله Private) بـ Cloudflare Pages، أمر البناء `npm run build` ومجلد الإخراج `dist`.
2. فعّل **Zero Trust → Access** وأضف سياسة (بريدك برمز تحقق) لحماية الرابط على مستوى الخادم.

بعد النشر: الرابط سرّي + محمي بكلمة مرور، وبياناتك تبقى على هاتفك.
